# MVP設計

目標: 30人の市場調査を通じて共通課題を発見し、まずは1院を無料導入して事例化する。

MVP要素:
- 市場調査による共通課題の仮説整理
- 1院無料導入の実行支援パッケージ
- 実行観察と改善ストーリーの記録
- 最初の事例としての事実ベースの成果報告

進め方:
1. 30人インタビューで課題仮説を整理
2. 仮説に合致する1院を選び、無料導入で伴走
3. 人＋テンプレ＋実行支援で現場改善を確認
4. 事例を得たら次は半自動化、そしてOS化の設計へ進む

モデル:
- 今は販売前提ではなく、最初の1事例のための無料導入
- 長期ビジョンは「人＋AI＋テンプレ→半自動化→OS化」
- 最初から大きなSaaSを作らない

期待成果:
- 1院での改善事例
- 課題マップと初期解決アプローチ
- 事例を基にした次のMVPの方向性
