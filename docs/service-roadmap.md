## Phase 1: 手動でGoogle Mapsから対象院を探す

**期間:** 1～2ヶ月

**対象:** ターゲット地域の整骨院

**アプローチ**
- Google Maps で直接検索（「整骨院」「接骨院」など）
- スクレッピングではなく、手動で確認
- 各院の情報を手作業でスコアリング

**実施内容**
- ターゲット地域：東京、大阪、名古屋など複数の都市
- 検索ワード：「整骨院」「接骨院」「カイロプラクティック」
- 手動抽出対象院：100～500院
- 各院の情報をスプレッドシートに記録
  - 院名、住所、電話、Webサイト有無、口コミ数、評価、写真数など

**成果目標**
- スコア-12～-8の改善機会高い院：100院特定
- 営業対象リスト作成
- 院長への営業メール素案作成

**パートナーの負担**
- 月80～120時間（手動作業 + データ集計）

**経営モデル**
- 固定費のみ（給与 or 外注）
- ROI測定：見込み客発見にかかった時間÷最終成約数

**移行条件（Phase 2へ）**
- 手動で100院以上を特定・スコアリング
- 営業メールの開封率 / 返信率を測定
- API自動化の必要性が確認できた

## Phase 2: 公式APIで候補院リストを作る

**期間:** 2～4ヶ月

**特徴**
- Google Maps Platform / Places API を統合
- スクレイピング不可（公式API利用）
- 大量の候補院を自動抽出

**実施内容**

### 技術実装
1. Google Maps Platform へ登録
   - Places API キーを取得
   - 請求設定（従量課金 or 月額制）
   - API リクエスト上限を確認

2. 検索機能の実装
   - 「整骨院」「接骨院」でキーワード検索
   - 地域別に複数クエリを実行
   - 各院の情報を自動取得
   - データベースに保存（規約内で）

3. 自動スコアリング機能
   - Webサイト有無：-3 or +0
   - 口コミ数（0～5件）：-2
   - 口コミ数（6～15件）：-1
   - 口コミ評価 < 3.5：-1
   - 写真数（0～5枚）：-1
   - 投稿3ヶ月以上なし：-2
   - 予約導線なし：-2
   - LINE導線なし：-1
   → スコア自動計算

4. 候補院リストの生成
   - スコア-12～-8：「最優先」として自動抽出
   - 地域ごと、スコアごとにグループ化
   - ダッシュボードで可視化

**成果目標**
- API連携完了
- 候補院リスト：500～2000院自動生成
- スコアリング精度：80%以上

**パートナーの負担**
- 開発：60～100時間（初期実装）
- 運用：月5～10時間（API監視、エラー処理）

**経営モデル**
- Google Maps Platform 費用：月$300～1,000（API従量課金）
- 開発パートナー：一括費用 or 継続月額
- ROI：Phase 1 比で自動化による工数削減

**移行条件（Phase 3へ）**
- API 連携の安定動作確認（1ヶ月以上）
- 候補院リスト 500 院以上で検証済み
- 初期 Phase 1 の営業結果と比較して精度確認

## Phase 3: 候補院ごとに簡易診断レポート自動生成

**期間:** 3～6ヶ月

**特徴**
- 各院の自動スコアリングから、簡易診断レポートを自動作成
- 営業メールの本文に組み込める
- 院長の心理的ハードル低下

**実施内容**

### レポート自動生成システム
1. テンプレート設計
   ```
   【院名】：XXX整骨院
   
   【Web集客強度スコア】
   -8/0 点（改善機会：大）
   
   【現在の課題】
   ✓ Google口コミ：3件 → 改善で月新患 50% 増が見込める
   ✓ Webサイト：未設置 → 検索からの流入ロス
   ✓ 予約導線：電話のみ → 24h 受付不可
   ✓ 投稿：3ヶ月以上なし → 信頼度低下
   
   【改善優先度TOP 3】
   1. Google口コミ獲得（3ヶ月で +12件目標）
   2. Webサイト or ランディングページ制作
   3. LINE導線確保 + 初回フォロー自動化
   
   【次のステップ】
   無料で詳しくご診断させます。
   無料ヒアリングはこちら → [Link]
   ```

2. レポート自動生成API
   - 各院のスコアから、診断レポートテキストを自動生成
   - 改善効果の見積もり（Aピラミッドモデル）
   - 優先度付け（ROI順）

3. 営業メール自動作成
   - テンプレート + 院ごとのレポート内容
   - 件名、本文、CTA（無料ヒアリングボタン）
   - 個人化度を高める

**成果目標**
- 自動レポート生成システム完成
- 営業メール 1000 通以上を自動作成
- 開封率：20～30% 目標
- 返信率：2～5% 目標（ヒアリング申し込み）

**パートナーの負担**
- 開発：80～120時間（レポート生成 + メール API）
- 運用：月10～15時間（結果測定、改善）

**経営モデル**
- 初期開発コスト：¥200～400 万
- 運用：月 $100～300（メール API 等）
- ROI：「見込み客 1 人あたり取得コスト」を下げる

**移行条件（Phase 4へ）**
- レポート精度：実際の院の課題と 80% 一致
- メール開封率：20% 以上
- 返信率：2% 以上（ヒアリング成約）

## Phase 4: 無料ヒアリングへ誘導

**期間:** 1～3ヶ月（Phase 3と並行）

**特徴**
- 簡易診断レポートから、「無料ヒアリング」への自然な誘導
- 営業感を排除（診断 + 提案という位置づけ）
- 約30分の無料相談で、実行支援への入口化

**実施内容**

### 無料ヒアリングの設計
1. **目的**
   - 簡易診断が実際の課題と一致するか検証
   - 院長の深い背景（スタッフ、経営、心理面）を理解
   - 実行支援への意欲度を測定

2. **フロー**
   ```
   簡易診断レポート（メール）
        ↓
   「詳しくご診断させませんか」
        ↓
   無料ヒアリング申し込み
        ↓
   Google フォーム / Calendly で日程予約
        ↓
   30分ヒアリング（電話 or Zoom）
        ↓
   「次のステップ」を提案
   （実行支援 or セルフでやる）
   ```

3. **ヒアリング質問項目**
   - docs/interview-questions.md を参照
   - 課題検証 + 深掘り質問
   - 実行支援の意欲確認

4. **ヒアリング後**
   - 改善提案書を作成（実行支援の詳細）
   - 「実行サポートを申し込む」 or 「セルフでやる」を選択
   - 成約率 30～50% を目指す

**成果目標**
- ヒアリング申し込み：500 件中 30～50 件（6～10%）
- ヒアリング実施率：80%（日程調整の成功率）
- ヒアリング後の成約率：30～50%

**パートナーの負担**
- ヒアリング実施：月 40～60 時間（30 件 × 30 分 = 15 時間）
- 提案書作成：月 20～30 時間
- フォローアップ：月 10～15 時間

**経営モデル**
- 初期実行支援契約：¥30～50 万（初期設定 + 1ヶ月サポート）
- 月額費用：¥5～15 万（進捗管理 + コンサル）
- 初期事例：2～3 院確定

**移行条件（Phase 5へ）**
- 初期事例 1～3 院の支援確定
- ヒアリングから成約への conversion rate が 30%+
- 初期事例の改善が 3 ヶ月で目に見える成果

## Phase 5: 実行支援を提供

**期間:** 6ヶ月（初期事例ごと）

**特徴**
- テンプレ + 伴走支援で、院長が実行できるように
- 月 1 回の進捗面談で改善サイクルを回す
- 初期事例の成功ストーリーを記録

**実施内容**

### 実行支援の内容
1. **初期セットアップ（1ヶ月目）**
   - Google ビジネスプロフィール最適化
   - LINE 公式アカウント設定
   - KPI ダッシュボード設定

2. **実行支援（2～6ヶ月目）**
   - 月 1 回の進捗面談
   - テンプレ + チェックリストで実行
   - 課題が出たら即座にサポート
   - 改善案の提案・実装

3. **成果測定**
   - 新患数：目標 30+（安定）
   - リピート率：目標 40%+
   - 月商：目標 200万円+
   - 毎月の成長トレンド可視化

### 初期事例のストーリー化
1. **支援前の状態を記録**
   - 新患数、リピート率、月商
   - Google 口コミ、写真、投稿
   - 院長の困り事

2. **支援の過程を記録**
   - 何をしたか
   - どうなったか
   - 院長のコメント

3. **成功ストーリーを発表**
   - ブログ記事 / YouTube
   - 業界メディアへの投稿
   - 次の営業に使用

**成果目標**
- 初期事例 1～3 院が 6 ヶ月で目標達成
- 新患：月 30+ 安定
- リピート率：40%+
- 「成功ストーリー」3 パターン完成

**パートナーの負担**
- 支援時間：月 15～25 時間/院（面談 + コンサル + 実行サポート）
- 初期事例 3 院の場合：月 45～75 時間
- ストーリー化：月 10～20 時間

**経営モデル**
- 初期手数料：¥30～50 万/院
- 月額費用：¥5～15 万/院
- 初期 3 院の場合：月 ¥15～45 万（経常利益）

**移行条件（Phase 6へ）**
- 初期事例が 6 ヶ月で目標達成
- 成功ストーリーが業界で認知開始
- テンプレの再現性が 80% 確認
- 次の 10 院への展開計画確定

## Phase 1: Person-to-Person（人力伴走）

**対象:** 開業直後の1～3院

**特徴**
- コンサルタント/パートナーが手取り足取りサポート
- オーダーメイド対応
- 成果ベースの料金体系

**提供物**
- Google最適化ガイド
- LINE運用テンプレ（基本20パターン）
- 月1回の進捗面談
- LINE/メール相談無制限

**院長の負担**
- 毎月2～4時間（実務時間）
- 提案された施策を実行するだけ

**パートナーの負担**
- 月8～16時間/院（支援時間）
- 面談、コンサル、実行指導

**成果目標**
- 新患：月5～10 → 月30～50
- リピート率：10% → 40%
- 月商：30万～100万 → 150万～300万
- 利益改善：明確化と20% increase

**期間:** 6ヶ月

**経営モデル**
- 初期手数料：¥30～50万（Google優化 + LINE設定）
- 月額費用：¥5～15万（進捗管理 + コンサル）
- インセンティブ：売上×2～5%

**移行条件（Phase 2へ）**
- 月新患が月30以上に安定
- リピート率が40%以上
- 「自分たちで改善を継続できる」という自信

## Phase 4: Web Service（Webサービス化）

**対象:** 30院以上の大規模ネットワーク

**特徴**
- 院長向けの「業界標準SaaS」になる
- AI・予測分析の導入
- 新患獲得～経営分析まで一元化

**提供物**
- 統合クラウドプラットフォーム
  - Google・LINE・予約システム統合
  - 患者管理の一元化
  - AI予測による施策提案
  - 自動ROI分析

- 新患獲得エンジン
  - Google自動最適化
  - 口コミ自動獲得フロー
  - LINE自動配信（AI最適化）

- 経営分析AI
  - 日次のKPI自動集計
  - 利益予測（AI学習）
  - 「次の施策は何？」という提案

- 業界プラットフォーム機能
  - 院長同士のネットワーキング
  - ジョブボード（スタッフ募集）
  - リソース共有マーケットプレイス

**院長の負担**
- ほぼゼロ（ダッシュボード閲覧のみ）

**パートナーの負担**
- 月10～30分/院（ツール運用）
- 新機能開発、ベストプラクティス更新

**成果**
- 業界のスタンダード化
- 多くの院が「集客×リピート×経営」で最適化

## リスク管理と柔軟性

### リスク認識
1. **技術的リスク** - ツール開発が遅延する
   - 対策：Phase 3までは人力 + Excel中心で継続
   - Phase 3開始時に外部開発パートナーと契約

2. **市場リスク** - 院長が機械化を抵抗する
   - 対策：成功事例をビデオ・ブログで大量発信
   - テンプレ段階で「自分たちの成功」を実感させる

3. **競合リスク** - 大手企業が参入する
   - 対策：小規模院特化で差別化
   - 早期に100院達成、スイッチングコスト構築

### 柔軟なアプローチ
- **各フェーズは「固定」ではなく「目安」**
- 市場反応、技術進化に合わせて調整
- 必要に応じて「Phase 3.5」など、中間段階追加
