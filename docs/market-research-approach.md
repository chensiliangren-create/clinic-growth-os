## 2段階の市場調査アプローチ

### アプローチ 1：Google Maps 公開情報からの課題仮説

**目的**
- スクリーニング：見込み客となる院を特定
- 課題の事前仮説：院長のWebサイトやGoogleビジネスプロフィールから、改善機会を予測

**手順**

1. **Google Maps / Places API で候補院を抽出**
   - 検索条件：「整骨院」「接骨院」など
   - 地域指定：ターゲット地域
   - 取得項目：名前、住所、電話、Webサイト、口コミ数、写真、投稿日

2. **各院をスコアリング**
   - Webサイト有無、口コミ数、評価、写真数、投稿頻度、予約導線、LINE導線
   - スコアリング基準は [docs/lead-generation-strategy.md](lead-generation-strategy.md) を参照

3. **改善機会を仮説化**
   - スコア -12～-8 の院 → 改善機会が大きい
   - 「この院は Google 口コミが 3 件。改善できれば月 50% の新患増加が見込める」
   - 具体的な改善ポイント 3 つを自動生成

4. **簡易診断レポート作成**
   - 院への営業資料として、改善仮説を提示
   - 「無料で、より詳しくご診断させていただきませんか」という誘導

**利点**
- ✓ 営業対象者を効率的に絞り込める
- ✓ 院長の課題を事前に把握できる（ヒアリングの準備）
- ✓ 営業メールがスパムにならない（具体的な改善提案）

**注意点**
- ✓ 公式 API 利用（スクレイピング禁止）
- ✓ 公開情報のみを使用（個人情報不可）
- ✓ 営業倫理を意識（1 回の提案で決める）

### アプローチ 2：院長への直接ヒアリング

**目的**
- 検証：Google Maps で仮説化した課題が、実際に院長の困り事と合致するか確認
- 深掘り：Google Maps では見えない課題（スタッフ、経営、心理面）を聞き取る
- 学習：共通パターンを発見

**判断基準**
「院長の言葉で課題を記録し、自分たちの想像で解釈しない。事例ベースの課題を集める。」

**前準備**
- Google Maps からの仮説レポートを準備
- 仮説に基づいたヒアリング質問を作成
- 確認型（閉じた質問）ではなく、深掘り型（開いた質問）を用意

### メール/電話共通の流れ

1. **挨拶と簡単な自己紹介**
2. **調査目的の説明**
   - 「Google Maps で拝見した貴院の情報から、改善機会を発見しました」
   - 「それが実際の課題と一致するか、現場の声を伺いたいです」
3. **協力依頼**
   - 「5～10分のヒアリングにご協力いただけますか？」
4. **日程調整**
   - オンラインか来院時の短時間インタビュー
5. **お礼**
   - 取材後に、仮説整理と簡単なレポートを提供する旨を伝える

## メールテンプレ（改善版）

```
件名：【Google Maps で発見】XXX整骨院の Web 集客改善機会のご提案

先生

突然のご連絡失礼します。

整骨院・治療院の経営課題を整理し、本当に役立つ支援サービスを作るための調査を行っています。

Google Maps を拝見したところ、貴院は以下の改善機会がありそうです：

✓ Google 口コミが 3 件（目標：15 件以上）
  → 改善できれば、新患の 50% 増加が見込める

✓ Webサイトが未設置
  → Google 検索からの流入が逃げている

✓ LINE 導線が見えない
  → リピート患者との接触機会を逃している

これらが実際の課題であるかどうか、5～10 分ほどお聞きしたいです。

また、無料で詳しいご診断も承ります。

ご協力いただけますでしょうか？

よろしくお願いいたします。
```

## ヒアリングのポイント

**確認型質問（Google Maps で仮説化した項目）**
- 「Google 口コミが少ないことで、困っていることはありますか？」
- 「Webサイトなしで、新患獲得に支障がありますか？」
- 「LINE を使った患者フォローが実現できていないのは、なぜですか？」

**深掘り型質問（背景や心理面）**
- 「新患が安定しない理由は何だと思いますか？」
- 「Google 口コミを増やすために、今まで試したことはありますか？」
- 「経営が難しい時期は、どの指標を確認していますか？」

**目的は課題発見であり、解決策の押し売りではない**
- まず事実を聞き取り、院長の言葉で課題を整理
- 共通パターンを見つけることを最優先とする
- ヒアリング終了後、院長が「実行支援してほしい」と言ったら、初めて詳細提案

## 調査データの活用

### 学習ループ

```
Google Maps スコアリング
       ↓
院長ヒアリング（仮説検証）
       ↓
共通課題の発見
       ↓
改善テンプレの設計
       ↓
初期事例の支援（実行）
       ↓
事例ストーリーの記録
       ↓
テンプレの改善 → (REPEAT)
```

### 記録方法

- 各ヒアリングを音声記録 + テキスト化
- 課題ごとに集計（新患獲得、リピート、経営管理など）
- 共通パターン抽出
- 例外・特殊ケースも注記
