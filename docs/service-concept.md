## 事業の入口：Web集客導線診断

### ビジネスモデル

1. **見込み客の自動発見**
   - Google Maps / Places API で、Webサイトが弱い整骨院を特定
   - スコアリング：Google口コミ数、サイト有無、予約導線、LINE導線
   - 改善機会が高い院をランク付け

2. **簡易診断レポート作成**
   - 院ごとに「今、こんなWeb集客上の改善機会があります」というレポートを自動生成
   - 「新患が50%増える可能性」といった具体的な改善見積もり
   - ツール販売ではなく、「診断」という価値を先に提供

3. **無料ヒアリングへの誘導**
   - 簡易診断レポートから「具体的な実行方法を無料で相談しませんか」と提案
   - 低リスク・高価値の入口を作る

4. **実行支援へ**
   - ヒアリング後、院が「実行したい」と判断したら、テンプレ + 伴走支援を提供

### 利用API

- **Google Maps Platform / Places API**（公式API利用）
- 個人情報取得なし
- 営業倫理遵守（迷惑営業にならない設計）
- API利用規約・保存可能項目を厳密に確認

## コアアイデア（既存）

- 最初から大きなSaaSを作るのではなく、まずは現場の声を丁寧に収集して課題を検証する。
- 売る価値はAIではなく、「患者数・リピート・経営改善の実務課題の解決」に集中する。

### 提供内容（段階的）

#### Phase 0：Web集客診断（入口）
- Google Maps API で対象院を自動発見
- 簡易診断レポート自動生成
- 無料ヒアリングへの誘導

#### Phase 1：ヒアリング＋実行支援
- 30分の無料ヒアリング
- 現状整理と課題把握
- 改善ポイント3つの提示
- 必要に応じて実行支援

#### Phase 2～：事例化＋テンプレ化
- 初期事例の成功ストーリー記録
- 改善テンプレの標準化
- 複数院への展開

#### Phase 3～：OS化
- 自動化・プラットフォーム化
- 業界スタンダード化

## 差別化要素

1. **既存サービスとの違い**
   - ✓ 営業から入るのではなく「診断」から入る
   - ✓ 「先生が気づいていない改善機会」を見つけて提示
   - ✓ Google Maps の公開情報を活用した非侵襲的なアプローチ
   - ✓ スクレイピングではなく公式API利用で信頼感確保

2. **院長にとっての価値**
   - ✓ 営業ではなく、現状診断が先（心理的ハードル低い）
   - ✓ 「無料で機会損失を見つけてくれる」という実務的価値
   - ✓ その後、実行支援も選べる（フリーミアム的）

3. **我々の戦略的価値**
   - ✓ 大量の見込み客を低コストで発見可能
   - ✓ 診断レポートから無料ヒアリングへの導線が自然
   - ✓ 初期事例を作りやすい（改善機会が高い院を狙える）

## 長期的発展（Clinic Growth OS へ）

現在のWeb集客診断は入口に過ぎず、最終的には以下を目指します：

- 集客（Google・口コミ・LINE） × リピート（予約・フォロー） × 経営（KPI可視化）
- を一元管理できるプラットフォーム化
- 業界スタンダード化による大規模スケール
